# AI Startup Red Flags (summary from analyses)

This document collects the main red flags surfaced by critical analyses of modern AI startups (sources: two YouTube critiques). Use this as a checklist to avoid reputational, legal, and product risks.

## Product / Technical
- **Superficial LLM wrapper:** Product merely skins existing LLM APIs without proprietary orchestration.
- **Replicable core tech:** Core user value can be reproduced by generic LLMs + simple scripts.
- **Feature over function:** Emphasis on flashy UI/demos rather than usable, robust features.
- **AI for subjective assessment:** Using AI to judge people (language fluency, personality) without transparency or evaluation.

## Business / Marketing
- **Exaggerated claims:** Promises like "10x productivity" or "eliminate bias" without solid evidence.
- **Ragebait / deceptive marketing:** Viral stunts, edited posts, or emotional manipulation to attract attention.
- **Questionable endorsements:** Listing big customers or results without verifiable proof.

## Privacy / Safety
- **Intrusive surveillance:** Recording or monitoring users/employees/customers without clear opt-in and safeguards.
- **Overbroad permissions:** Requesting broad access (files, mail, screen) when minimal access would suffice.
- **Unclear retention & sharing:** Not stating how long data is kept, who can access it, and where it is stored.

## UX / Workflow
- **Opaque agent actions:** Agents perform high‑impact actions (emails, posting, file edits) without clear preview or undo.
- **No audit / logs:** Users cannot review historical agent decisions or automation runs.
- **Non-dismissable overlays:** Pop-ups that are intrusive or hard to close, degrading user control.

## Culture / Org
- **Hustle culture branding:** Glorifying overwork, dismissing critics; cult-like messaging.
- **Dismissive of privacy/ethics concerns:** Minimizing or ignoring user concerns about data usage.

---

## Avoidance Checklist (short)
- Local-first defaults; cloud sync opt-in with explicit permission.
- Granular per-integration permissions and clear consent UI.
- Auditable agent actions, preview/dry-run, and undo for destructive tasks.
- Short retention windows, end-user data controls (view/edit/delete), encryption in transit & at rest.
- Honest, reproducible demos and evidence-backed claims.
- Clear, plain-language privacy policy and a short security FAQ.
