# PKCE (Proof Key for Code Exchange) — Overview for Stuard AI (Web)

This document explains PKCE and how it's used with Stuard AI web and desktop authentication to Microsoft OAuth / Microsoft Graph.

---

## What is PKCE?
PKCE (Proof Key for Code Exchange) is an OAuth 2.0 extension that secures the authorization code grant by binding the authorization code to a one-time secret generated by the client. It prevents authorization code interception attacks and is essential for public clients.

## When to use PKCE
• Public clients (desktop, mobile, SPAs) — use PKCE.  
• Confidential clients (server-side web apps) may not need PKCE if they use client secrets, but PKCE is harmless and can be used for consistency.

## How PKCE works (brief)
1. Client generates `code_verifier` (random).  
2. Client computes `code_challenge = BASE64URL(SHA256(code_verifier))`.  
3. Client includes `code_challenge` in the authorize request.  
4. Authorization server returns `code`.  
5. Client posts token request including `code_verifier`.  
6. Server validates and issues tokens.

## Why Desktop + Web use it together
• The desktop app (public client) uses PKCE to securely obtain tokens without a client secret.  
• The web app (confidential client) performs server-side token exchange using a client secret and can also accept PKCE for consistency.  
• Use the same Azure App Registration with both redirect URIs to share consent and permissions.

## Best practices
- Use `S256` method.  
- Use high-entropy verifiers (>=32 bytes).  
- Keep `code_verifier` in memory only until exchange finishes.  
- Store refresh tokens securely on desktop; server-side storage for web tokens.
